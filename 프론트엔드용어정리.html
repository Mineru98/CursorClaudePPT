개요

프론트엔드 개발 분야에서 자주 사용되는 61가지 핵심 용어를 알파벳 순서로 정리했습니다.

이 프레젠테이션은 다음 내용을 다룹니다:

  • 기본 웹 개발 용어
  • 프론트엔드 프레임워크 및 라이브러리 관련 용어
  • 성능 및 최적화 관련 용어
  • 디자인 및 사용자 경험 관련 용어
  • 개발 도구 및 프로세스 관련 용어

주요 용어 카테고리

기본 웹 개발 용어

HTML

Hypertext Markup Language는 웹사이트를 구축하기 위해 사용되는 기본 마크업 언어입니다.

CSS

Cascading Style Sheets는 HTML 요소의 스타일과 레이아웃을 정의합니다.

JavaScript

웹 페이지에 상호작용과 동적 기능을 추가하는 클라이언트 사이드 스크립트 언어입니다.

DOM

Document Object Model은 HTML 문서를 트리 구조로 표현하는 프로그래밍 인터페이스입니다.

프레임워크 및 라이브러리

프레임워크 vs 라이브러리

프레임워크와 라이브러리는 개발 속도를 높이는 도구이지만 중요한 차이가 있습니다.

프레임워크 라이브러리
전체 애플리케이션 구조 제공 특정 기능에 집중된 도구 모음
제어의 역전(IoC) 원칙 적용 개발자가 직접 제어
예: Angular, Vue.js 예: React, jQuery

성능 및 최적화 용어

FCP (First Contentful Paint)

브라우저가 DOM 콘텐츠의 첫 번째 부분을 렌더링하는 데 걸리는 시간을 측정합니다.

LCP (Largest Contentful Paint)

가장 큰 콘텐츠 요소가 화면에 렌더링되는 데 걸리는 시간을 측정하는 성능 메트릭입니다.

경량화 (Minification)

파일 크기를 줄이기 위해 코드와 마크업을 최소화하는 과정입니다.

캐시 (Cache)

웹사이트 속도를 높이기 위해 관련 정보를 저장하는 임시 데이터 저장소입니다.

디자인 및 사용자 경험 용어

UI (User Interface)

사용자와 컴퓨터 간의 상호 작용 인터페이스입니다.

UX (User Experience)

사용자의 경험이 얼마나 만족스럽고 성공적인지에 집중한 UI와 사용자의 상호 작용을 의미합니다.

반응형 디자인 (Responsive Design)

사용자가 어떤 장치에서 웹사이트를 보던지 관계없이 웹사이트가 올바르게 보이도록 합니다.

모바일 퍼스트 (Mobile-first)

웹사이트를 구축할 때 데스크톱보다 모바일 기기를 우선시하는 웹 디자인 및 개발에 대한 접근법입니다.

개발 도구 및 프로세스

Git

개발자가 코드를 저장하고 관리할 수 있는 버전 관리 시스템입니다.

GitHub

Git을 위한 클라우드 인터페이스로, 버그 추적, 작업 관리 및 프로젝트 위키와 같은 기능을 제공합니다.

CI/CD

앱 개발 단계에 자동화를 도입해 고객에게 앱을 수시로 전달하는 방식입니다.

NPM/Yarn

JavaScript 패키지 관리자로, 프로젝트 의존성을 관리합니다.

프론트엔드 용어 사용 빈도

결론

프론트엔드 개발 분야는 빠르게 발전하고 있으며, 새로운 용어와 기술이 계속 등장하고 있습니다.

이러한 핵심 용어들을 이해하는 것은:

  • 팀 내 의사소통 개선
  • 기술 문서 이해도 향상
  • 개발 효율성 증가
  • 최신 트렌드 파악 용이

자주 묻는 질문

Q: 프론트엔드 개발자가 되기 위해 모든 용어를 알아야 하나요?

A: 모든 용어를 완벽히 알 필요는 없지만, 기본적인 용어들은 이해하고 있어야 효과적인 개발과 의사소통이 가능합니다.

Q: 이 용어들은 얼마나 자주 업데이트되나요?

A: 프론트엔드 개발 분야는 빠르게 변화하므로, 매년 새로운 용어들이 등장하고 일부는 사용되지 않게 됩니다.

Q: 초보자가 먼저 배워야 할 용어는 무엇인가요?

A: HTML, CSS, JavaScript, DOM, API, 반응형 디자인과 같은 기본 용어부터 시작하는 것이 좋습니다.
