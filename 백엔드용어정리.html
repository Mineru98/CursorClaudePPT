개발 용어의 중요성

개발 과정에서 마주치게 되는 다양한 용어들을 이해하는 것은 프로젝트를 원활하게 진행하는 데 큰 도움이 됩니다.

API

애플리케이션 간 상호작용을 가능하게 하는 인터페이스

서버

데이터와 서비스를 제공하는 컴퓨터 시스템

데이터베이스

체계적인 데이터 저장 및 관리 시스템

이 프레젠테이션에서는 백엔드 개발의 핵심 용어를 쉽게 이해할 수 있도록 설명합니다.

1. API (Application Programming Interface)

애플리케이션 프로그래밍 인터페이스는 소프트웨어나 프로그램이 서로 상호작용할 수 있도록 도와주는 도구입니다.

API는 다른 시스템이나 애플리케이션의 기능을 사용할 수 있게 해주는 '메신저' 역할을 합니다.

API 동작 예시

날씨 앱

사용자가 날씨 정보 요청

날씨 API

요청을 처리하고 데이터 중개

날씨 서버

날씨 데이터 제공

API 관련 주요 용어

Endpoint

API가 호스팅되는 서버의 특정 URL 주소입니다.

https://api.example.com/users

Request & Response

클라이언트의 요청과 서버의 응답을 나타냅니다.

Request: 정보 요청
Response: 서버 응답

HTTP Methods

웹에서 데이터를 주고받는 방법을 정의합니다.

GET: 데이터 조회
POST: 데이터 생성
PUT: 데이터 수정
DELETE: 데이터 삭제

JSON

데이터 교환을 위한 경량 포맷입니다.

{
  "name": "홍길동",
  "age": 30,
  "role": "개발자"
}

REST

Representational State Transfer, 웹 API 설계를 위한 아키텍처 스타일입니다.

자원 기반 상태 비저장 표준 HTTP 메소드 다양한 표현

2. 서버(Server)

네트워크 상에서 데이터나 서비스를 제공하는 컴퓨터나 소프트웨어 시스템입니다.

사용자가 웹 브라우저나 애플리케이션을 통해 정보를 요청할 때, 서버는 그 요청을 받아 처리하고 필요한 데이터를 사용자에게 돌려줍니다.

서버-클라이언트 모델

서버의 역할

  • 데이터 저장 및 관리
  • 클라이언트 요청 처리
  • 비즈니스 로직 실행

클라이언트의 역할

  • 사용자 인터페이스 제공
  • 서버에 요청 전송
  • 응답 데이터 표시

서버 관련 주요 용어

호스팅 (Hosting)

인터넷 상에서 웹사이트나 웹 애플리케이션을 사용할 수 있도록 서버 공간을 제공하는 서비스입니다.

클라우드 호스팅 공유 호스팅 VPS

IP 주소 (IP Address)

인터넷상에서 컴퓨터나 다른 장치를 식별하는 데 사용되는 고유한 번호입니다.

IPv4
192.168.1.1
IPv6
2001:0db8:85a3:0000:0000:8a2e:0370:7334

포트 (Port)

컴퓨터나 서버에서 특정 서비스에 대한 접근 지점을 나타내는 숫자입니다.

HTTP: 80
HTTPS: 443
FTP: 21
SSH: 22

프로토콜 (Protocol)

컴퓨터나 장치들이 서로 통신할 때 지켜야 할 규칙이나 표준입니다.

HTTP/HTTPS
웹 통신 프로토콜
TCP/IP
인터넷 기본 프로토콜
SMTP
이메일 전송 프로토콜
FTP
파일 전송 프로토콜

로드 밸런싱 (Load Balancing)

네트워크 트래픽이나 요청을 여러 서버에 고르게 분산시켜 각 서버에 가해지는 부하를 분산시키는 기술입니다.
